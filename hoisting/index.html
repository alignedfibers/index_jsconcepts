Variable hoisting in JS Description
When javascript evaluates a function, it moves the declarations of all variables and functions to the top of the function. This is called "Hoisting". When a variable is hoisted, it's assignment and declaration are split in two, such that all variables within a function are initialized to undefined. Variables will be assigned values in the order the assignments appear within the function as it is run. This can cause some confusion because late declared vars are not captured in the TryCatch with the error of Undeclared, and a presumption of block scope carried over from other languages can cause infinite loops, and logic that does not evaluate as expected. It is recommended to declare variables just after the opening curly brace within your function, nested functions are hoisted as well. You should define functions, and assign values to variables before they are used.
